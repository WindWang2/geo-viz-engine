# 测井可视化引擎 V2 设计方案 (Geoviz Engine V2)

本项目旨在基于 `demo.jpg` 和 `demo.xls` 提供的专业地质标准，完全重新设计测井可视化引擎。

## 1. 核心目标
- **1:1 视觉还原：** 参照 `demo.jpg` 实现高度专业的测井综合解释图。
- **地图集成：** 使用 OpenStreetMap (OSM) 底图展示井位点。
- **数据驱动：** 完全对接 `demo.xls` 中的多 Sheet 复杂数据（地层、岩性、曲线、相、层序）。
- **专业符号库：** 实现 SVG 矢量地质符号库。

## 2. 架构方案

### 2.1 地图模块 (WellMap)
- **底图引擎：** MapLibre GL JS。
- **图层：** OSM 栅格切片底图 + 矢量测井井位图层。
- **交互：** 点击井位标记通过 `react-router` 跳转至详情页 `/well/:id`。

### 2.2 综合图表模块 (WellLogDashboard)
- **布局技术：** **CSS Grid**。定义 17 列固定比例网格，确保表头对齐。
- **表头 (Header)：**
  - 多级嵌套表头（地层系统、沉积相、三级层序）。
  - 动态显示曲线量程（如 AC: 70-40, GR: 0-200）。
- **内容轨道 (Tracks, 17列)：**
  - **地层系统 (col 1-4)：** 系、统、组、段，支持跨深度合并单元格及垂直文字。
  - **测井曲线道 (col 5)：** AC/GR/DEN，使用 `<svg>` 和 `<path>` 渲染高频数据。
  - **深度道 (col 6)：** 黑色粗体深度标尺（如 2515, 2520...）。
  - **取心道 (col 7)：** 取心位置标记（Placeholder）。
  - **岩性道 (col 8)：** 背景色 + SVG Pattern 填充。自动匹配”砂岩”、”泥岩”等关键字。
  - **照片道 (col 9)：** 岩心照片占位（Placeholder）。
  - **测井曲线道 (col 10)：** RT/RXO/NPHI。
  - **岩性描述 (col 11)：** 文本换行显示。
  - **沉积相 (col 12-14)：** 微相、亚相、相，深度区间着色或文字居中显示。
  - **三级层序 (col 15-17)：** 层序结构、体系域、层序。

### 2.3 后端数据映射 (Python API)
- **数据处理：** 使用 `Pandas` 读取 `demo.xls`。
- **数据整合：** 将散落在不同 Sheet 的数据按“井号”和“深度”进行关联对齐。
- **数据格式：** 返回结构化的 JSON，`WellLogData` 包含 `metadata`（井名、坐标、深度范围）、`curves`（采样曲线数据）、`intervals`（区间数据：地层系统、岩性、沉积相、层序）。`intervals` 字段为 Optional，前端使用 `EMPTY_INTERVALS` 常量作为 fallback。

## 3. 技术难点与解决

- **SVG 符号库：** 定义一套标准的 `<pattern>` 库。支持“复合岩性”（背景色+Pattern）。
- **长页面性能：** 测井图可能跨越数千米，使用 CSS Grid 配合原生的 `overflow-y-auto`，仅在必要时引入虚拟滚动。
- **比例缩放：** 提供深度比例尺调节功能（如 1:200, 1:500）。

## 4. 成功标准
- 浏览器渲染出的图表在布局、颜色、文字排布上与 `demo.jpg` 视觉一致。
- 能够动态加载并正确显示 `demo.xls` 中“老龙1”井的所有数据。
- 地图能够正确显示坐标位置并支持点击跳转。
