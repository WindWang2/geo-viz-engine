**Release Date:** 2026-05-12
**Tier:** Standard

## Overview

This release focuses on hardening the engine's security posture and validating the performance gains of the modular architectural transition. Key highlights include the removal of insecure serialization and the formalization of GPU-accelerated seismic slicing.

## 🛡️ Security Hardening

### Arbitrary Code Execution (RCE) Mitigation
- **Vulnerability**: The previous caching mechanism used Python's `pickle` module, which is susceptible to RCE if an attacker provides a crafted `.pkl` file.
- **Fix**: Migrated the entire caching layer to **Pydantic-based JSON serialization**. The engine now uses `model_validate_json` for safe, schema-validated loading.
- **Verification**: `src/data/loaders.py` now uses `.json` extensions and pure JSON data. All residual `.pkl` files were purged.

### Data Privacy & Git Hygiene
- **Issue**: Real oil/gas well coordinates were being tracked in the repository.
- **Fix**: Removed `data/well_coordinates.json` from git tracking and updated `.gitignore`.
- **Added**: `data/well_coordinates.example.json` provided for external contributors.

### WebView Security
- **Mitigation**: Added a secure wrapper for `QWebEngineView` in the PaleoMap module. It blocks unauthorized navigation attempts, preventing potential local file exfiltration through JS injection.
