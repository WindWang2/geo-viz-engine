GeoViz Engine — Architecture Options

推荐

方案A：Tauri + 嵌入式Python

Python 打包进应用，单进程运行
桌面壳
Tauri 2.x Rust后端 · 2-5MB · WebView渲染
前端渲染
D3.js 曲线/等值线 · deck.gl 大规模渲染 · Three.js 3D
Python后端（嵌入式）
FastAPI · uvicorn 内嵌启动 · PyInstaller 打包分发
本地 localhost 通信 · 无需外部安装Python
数据处理
lasio 测井 · segyio 地震 · GDAL 栅格矢量 · scipy 插值等值线

优势

  • 单机部署简单，双击运行
  • Python生态完整可用
  • 支持局域网模式（改为监听0.0.0.0）
  • Tauri轻量，适合地质工作站

风险

  • PyInstaller打包体积较大(~200MB)
  • Python嵌入增加了构建复杂度
  • Rust侧需写通信桥接代码
★ 推荐：单机体验最好，无需外部依赖
备选

方案B：Tauri + 外置Python服务

Python独立运行，应用连接服务
桌面壳
Tauri 2.x Rust后端 · 2-5MB · WebView渲染
前端渲染
同方案A：D3.js · deck.gl · Three.js
Python后端（独立进程）
FastAPI · uvicorn 独立运行
需要用户自行安装 Python 3.11+
支持局域网/服务器部署
数据处理
同方案A：完整Python地质数据生态

优势

  • 架构最清晰，前后端完全解耦
  • 局域网/云端部署零修改
  • 打包体积小（不含Python）
  • Python端可独立升级

风险

  • 用户需自行安装Python环境
  • 需要管理Python进程启停
  • 版本兼容性问题（用户Python版本不一致）
适合团队协作/局域网场景，但终端用户体验不如方案A

方案C：混合模式

开发时外置Python，发布时嵌入打包
桌面壳
Tauri 2.x · 开发/发布双模式
前端渲染
同上：D3.js · deck.gl · Three.js
Python后端（自适应）
开发时：venv 外置运行
发布时：PyInstaller 嵌入打包
通过环境变量 GEOVIZ_MODE 切换
数据处理
同上：完整Python地质数据生态

优势

  • 开发体验最好（热重载、快速调试）
  • 发布时打包，终端用户无需配置
  • 局域网模式也支持（外置Python）
  • 最灵活，两全其美

风险

  • 需维护两套启动脚本
  • 打包和发布流程更复杂
  • CI/CD需要覆盖两种模式测试
★ 实际最实用：开发用外置，发布用嵌入
